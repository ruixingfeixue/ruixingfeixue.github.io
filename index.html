css基本视觉格式化

发表于 2017-05-27 | 分类于 css权威指南第三版
行内布局基础术语

匿名文本：
是指所有未包含在行内元素中的字符串。
在标记<p>I'm<em>so</em>happy</p>中，序列”I’m”和”happy”都是匿名文本。
注意：空格也是匿名文本的一部分，因为空格和其它元素一样都是字符。

em框:
在字体中定义。也成为字符框(character box)。实际的字形可能比其em框更高或更矮。
在css中，font-size的值决定了各个em框的高度。

内容区:
在非替换元素中，内容区可能有两种：css2.1规范允许用户代理选择其中任意一种。
内容区可以是元素中各字符的em框串在一起构成的框，也可以是由元素中字符字形描述的框。
在替换元素中，内容区就是元素的固有高度再加上可能有的外边框，边框或内边距。

行间距:
行间距(leading)是font-size值和line-height值之差。这个差实际上要分为两半，分别应用到内容区的顶部和底部。
为内容区增加的这两部分分别称为半间距(half-leading)。行间距只适用于非替换元素。

行内框:
这个框通过向内容区增加行间距来描述。
对于非替换元素，元素行内框的高度刚好等于line-height的值。
对于替换元素，元素行内框的高度恰好等于内容区的高度，因为行间距不应用到替换元素。

行框:
这是包含该行中出现行内框的最高点和最低点的最小框。
换句话说，行框的上边界要位于最高行内框的上边界，而行框的底边要放在最低行内框的下边界。

行为和有用的概念：

  • 内容区类似于一个块级元素的内容框。
  • 行内元素的背景应用于内容区及所有内边距。
  • 行内元素的边框要包围内容区及所有内边距和边距。
  • 非替换元素的内边距、边框和外边距对行内元素或其生成的框没有垂直效果，
    也就是说，它们不会影响元素行内框的高度。(也不会影响包含该元素的行框的高度)
  • 替换元素的外边距和边框确实会影响该元素行内框的高度，相应的，也可能影响包含该元素的行框的高度。
  • 行内框在行中根据其vertical-align属性值垂直对齐。

css文本属性

发表于 2017-05-26 | 分类于 css权威指南第三版
行高

line-height属性是文本行间基线之间的距离。
line-height控制了行间距，这是文本行之间超出字体大小的额外空间。
换句话说，line-height值和字体大小之差就是行间距。

构造文本行

文本行的每个元素都会生成一个内容区，这由字体的大小决定。
这个内容区则会产生一个行内框(line-box)。
要确定一个给定元素的行间距，只需将line-height的计算值减去font-size的计算值。
这个值是总的行间距。


举个栗子：
假设font-size为14px(相应的，内容区的高度也是14px)，而且line-height为18px。
其差(4像素)除以2，将其一半应用到内容区的顶部和底部。这会得到一个18像素高的行内框。
在内容区的上面和下面分别有2个额外的像素。

指定line-height值

line-height的可取值，如果使用默认值normal，用户代理必须计算行间的垂直空间。
不同的用户代理计算出来的值可能不同，不过通常都是字体大小的1.2倍。
这使得行框要高于给定元素的font-size值。

垂直对齐文本

CSS的属性vertical-align用来指定行内元素(inline)或表格单元格(table-cell)元素的垂直对齐方式。

  1. vertical-align:baseline
    要求一个元素的基线与其父元素的基线对齐。
    如果一个垂直对齐元素没有基线-比如是一个图像或者表单输入元素-那么该元素的底端与其父元素的基线对齐。
  2. vertical-align:sub
    声明会使一个元素变成下标，这意味着其基线相对于其父元素的基线降低。
    规范并没有规定降低的距离，所以对于不同的用户代理，可能距离会有所不同。
  3. vertical-align:super
    声明会使一个元素变成上标，这意味着其基线相对于其父元素的基线升高。
    规范并没有规定升高的距离，所以对于不同的用户代理，可能距离会有所不同。
    注意：sub和super不会改变元素的字体大小。所以下标和上标文本不会变小或变大。所有文本都与父元素的文本大小相同。
  4. vertical-align:bottom
    将元素行内框的底端与行框的底端对齐。
  5. vertical-align:middle
    会把行内元素框的中点与父元素基线上方0.5ex处的一个点对齐。这里的1ex相对于父元素的font-size定义。
  6. vertical-align:top
    将元素行内框的顶端与包含该元素的行框的顶端对齐。
    7、vertical-align:text-top
    将元素行内框的顶端与包含该元素的内容区的底端对齐。
    8、<percentage>
    将元素上移或下移一定距离，这个距离由相对于元素line-height值指定的一个百分数确定。

css选择器和字体

发表于 2017-05-12 | 分类于 css权威指南第三版

本书是css权威指南第三版,涵盖了CSS2和CSS2.1(截止2006年4月11日发布的工作草案)。

概念

1.替换元素
替换元素是指用来替换元素内容的部分并非由文档内容直接表示。
例如img元素，它是由文档本身之外的一个图像文件来替换。
<img src="a.png"这个标记片段不包含任何具体内容，只有一个元素名和属性。
只有指定了src，否则没有任何意义。还有input也是替换元素，其类型取决于type。
2.非替换元素
大部分的html和xhtml都是非替换元素。其内容在元素本身生成的框中显示。

属性

link标签：rel代表关系(relation).值为stylesheet。media属性，css2为这个属性定义了很多可取值。
all 用于表现所有媒体
aural 用于语音合成器、屏幕阅读器和文档的其它声音表现。
浏览器的支持性自己去查啦。毕竟用的也不多。
等等。。

选择器

1.分组选择器
h2,p{color:grey}这样就将h2和p分到一个组啦。h2和p颜色都是灰色啦。

2.多类选择器
.warning.urgent{color:grey} 这样就可以给元素增加样式class="warning urgent"。元素既紧急又有警告了。
.warning.help{color:red} 这个选择器只匹配class包含warning、help的元素。

3.属性选择器
p[class~="warning"]{font-weight:bold}; 这样就可以选择class属性中包含warning的元素。
[foo^="ch"] 选择foo属性值以”ch”开头的所有元素
[foo$="le"] 选择foo属性值以”le”结尾的所有元素
[foo*="l"] 选择foo属性值包含子串”l”的所有元素
[lang|='en'] 选择foo属性值等于en或者以en-开头的元素

4、选择子元素
h1>strong {color:red;} 选择h1元素的第一个strong元素。第二个不受影响。

5、选择相邻兄弟元素
h1+p{ margin-top:0} 选择紧连在一个h1元素后出现的所有段落,h1要与p元素有共同的父元素。
注意：一个选择符只能选择两个相邻兄弟中的第二个元素。
li+li{font-weight:bold} 会把第二个和第三个li字体变粗。

6.伪类选择器
静态伪类：第一次显示之后不会再改变文档的样式，例如:link、:visited
动态伪类：可以根据用户行为改变文档的外观。例如:hover、:active、:focus
选择第一个子元素：:first-child来选择元素的第一个子元素

如代码，元素的第一个子元素包括第一个p，第一个em，第一个li。
p:first-child{ font-weight:bold;}
li:first-child{ text-transform:uppercase;}
第一个规则将作为某元素的第一个子元素的所有p元素设置为粗体。
第二个规则将作为某元素的第一个子元素的所有li元素变成大写。

5.伪元素选择器
:first-letter{color:red;} 用于设置一个块级元素首字母的样式。
:first-line{ color:red;} 用来影响元素中的第一个文本行。

注意点: :first-letter和:first-line只能运用于标记或者段落之类的块级元素，而不能运用于超链接等行内元素。

元素继承性

不仅正常的h1会变成灰色，连em文本也会变成灰色。因为继承的值没有特殊性，甚至连0特殊性都没有。而通配符选择器适用于所有元素，而且有0特殊性。其颜色声明指定的值gray要优先于继承值。

字体大小和继承

在CSS中font-size是可以继承的。不过继承的是计算值而不是百分数。

strong元素的大小值就算如下：
12px × 120% =14.4px ≈ 14px
14.4px × 135% = 19.44px ≈ 19px

字体变形

font-varient有两个非继承值:默认值normal和small-caps。
normal描述正常文本。small=caps要求使用小型大写字母文本。

视口和屏幕尺寸

视口和屏幕尺寸不是同一视口是指浏览器窗口内的内容区域，不包含工具栏、标签栏等。也就是网页实际显示的区域。个概念。

屏幕尺寸指的是设备的物理显示区域。
需要注意的是有些浏览器调整工具显示的尺寸包含浏览器的其他元素，诸如地址栏、标签栏和搜索栏，而有些工具则不是这样。

媒体查询

<link rel="stylesheet" media="screen and (orientation: portrait)" href="portrait-screen.css" />
媒体查询表达式询问了媒体类型（你是一块显示屏吗？），然后询问了媒体特性（显示屏是纵向放置的吗？）。
任何纵向放置的显示屏设备都会加载 portrait-screen.css样式表，其他设备则会忽略该文件。
在媒体查询的开头追加not则会颠倒该查询的逻辑。

限制只有视口宽度大于800像素的显示屏设备才能加载文件。

有两点需要注意。
第一，媒体查询之间使用逗号分隔。
第二，你会注意到在projection之后，没有and，也没有任何特性/值的组合。
没有后续表达式，意味着只要是projection就满足条件。本例中，样式会应用于所有的投影仪。

在屏幕宽度小于等于400像素的设备上,h1元素的文字颜色就会变成绿色。
@import url("phone.css") screen and (max-width:360px);

CSS的@import指令在当前样式表中按条件引入其他样式表。
如上代码会给视口最大宽度为360像素的显示屏设备加载一个名为 phone.css的样式表。
切记，使用CSS的@import方式会增加HTTP请求(这会影响加载速度),所以请谨慎使用该方法。

穿衣搭配系列六-形态的动静

发表于 2017-04-27 | 分类于 穿衣搭配

之前我们学些了款式的量感，轮廓的直曲。再来复习下。
量感其实就是判断服饰的成熟度，服饰的直曲是女性化和中性化成都度量的指标，曲代表女性化程度很强，直代表偏中性化的服饰。
例如条纹格纹字母，抽象的图案。越接近我们自身的成熟度和女性化程度的衣服越适合自己。

今天我们来学习服饰形态的动静判断。其实就是视觉印象上的均衡与否，冲击力，表现性容。
形态的动静 也是从色彩、款式和面料三个纬度去判断

款式

动：时尚款，外形奇特不和常规，内部装饰多。与传统相对，有明显变化的款式混搭，多层次搭配，装饰明显，裁剪特殊。设计感强，不同款式间的搭配变化多。
静：基本款，外形常规，内部装饰图案少或者是无图案。传统款式，基本款式，简洁，均衡，变化少。装饰性弱。

分细节来说就是
外廓型：对服装结构的解构，异于常规的轮廓涉及（衬衣扣子设计是反的，跟常规不一样）
内部装饰：越抓住眼球越动（与流行装饰关系不大，例如铆钉、亮片）
图案：来源于图案本身、分散、细碎，不稳定，变化感强。而条纹格纹有稳定感。

说人话就是
服饰抓眼球、特别的程度。

色彩

纯度：也叫色彩的饱和度，是颜色的含色量。越鲜艳的颜色，纯度越高，色彩含量越高；
越柔和的颜色，纯度越低，色彩的含量月低。所有的无彩色都是纯度为0的颜色。

动：鲜艳，纯度高，对比强
静：纯度比较低，对比弱

Q：冬天着装偏深色，那在配饰或者打底上选择亮色或者浅色搭配，是不是就整体变为动了？
A：黑白配(极端颜色配比，一半一半的时候对比度最强),如果黑中一点点白色，那就不是动。量感直曲动静都是相互联系的

面料

动：反光，特殊肌理，高科技，面料之间有强烈的对比
静：亚光，常规，同一种面料缺少对比

注意：只要是色彩、面料、款式有一个方面是动的，那么这件衣服整体的形态就是动的。

直曲居中内装饰：小圆领
直曲居中图案：无图案
就算是直筒形的外廓，如果有荷叶边或者花朵藤蔓等女性感强的图案，也是偏曲。

常用的hexo指令

发表于 2017-03-12 | 分类于 hexo

init

$ hexo init [folder]
新建一个网站。如果没有设置 folder ，Hexo 默认在目前的文件夹建立网站。

new

$ hexo new [layout] <title>
新建一篇文章。如果没有设置 layout 的话，默认使用 _config.yml 中的 default_layout 参数代替。如果标题包含空格的话，请使用引号括起来。

generate

$ hexo generate
生成静态文件。

选项 描述
-d, –deploy 文件生成后立即部署网站
-w, –watch 监视文件变动
该命令可以简写为

$ hexo g

publish

$ hexo publish [layout] <filename>
发表草稿。

server

$ hexo server
启动服务器。默认情况下，访问网址为： http://localhost:4000/。

选项 描述
-p, –port 重设端口
-s, –static 只使用静态文件
-l, –log 启动日记记录，使用覆盖记录格式

deploy

$ hexo deploy
部署网站。

参数 描述
-g, –generate 部署之前预先生成静态文件
该命令可以简写为：

$ hexo d

clean

$ hexo clean
清除缓存文件 (db.json) 和已生成的静态文件 (public)。
在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令。

version

$ hexo version
显示 Hexo 版本。

更多指令请参考 Hexo指令

在声明权重方面要考虑5级。权重由大到小的顺序依次为：
1、读者的重要声明
2、创作人员的重要声明
3、创作人员的正常声明
4、读者的正常声明
5、用户代理声明

这种情况下第二个规则会胜过导入的规则，因为第二个规则是文档样式表的一部分。

css每个选择器都有其特殊性。如果一个元素有两个或者多个冲突的属性声明，那么有最高特殊性的声明就会胜出。

a-行内样式 1000
b-ID选择器的数量 100
c-类、伪类和属性选择器的数量 10
d-标签选择器和伪元素选择器的数量 1
通配选择器对特殊性没有贡献。其特殊性为0。

value=a*1000+b*100+c*10+d

优先级高的覆盖优先级低的，后面的覆盖前面的，不同属性会合并。

替换元素： 据CSS2.1规范所述：“替换元素的特征在于，其内容超出了CSS格式化模型的范畴，比如图片、嵌入的文档或小应用程序等。”。
原则上我们无法为替换元素添加生成性内容，尽管某些浏览器可能会允许这样做。

然后，就可以把真正的复选框隐藏起来（但不能把它从tab键切换焦点的队列中完全删除），再把生成性内容美化一番，用来顶替原来的复选框！

这个创意的名字叫做“复选框hack”

通过阴影来弱化背景

一、box-shadow方案
(Demo点这里)
使用大外延值实现 box-shadow: 0 0 0 999px rgba(0,0,0,.8);
这个初步的解决方案有一个明显的问题，就是它无法在较大的屏幕分辨率（>2000px）下正常工作。
视口单位: vmax。1vmax相当1vw和1vh两者中的较大值。
100vw等于整个视口的宽度，100vh就是视口的高度。因此，满足我们需求的最小值就是50vmax 。
由于投影是同时向四个方向扩展的，这个遮罩层的最终尺寸将是100vmax 加上元素本身的尺寸。
酱汁：box-shadow: 0 0 0 50vmax rgba(0,0,0,.8);

缺点：
1.由于遮罩层的尺寸是与视口相关，而不是与页面相关的，当我们滚动页面时，遮罩层的边缘就露出来了，
除非给它加上 position: fixed;这个样式，或者页面并没有长到需要滚动的程度。
此外，由于页面很可能真的很长，为了规避这个缺陷而扩大投影的扩张半径就不太明智了。
相反，我推荐有限度地应用这个技巧，比如配合固定定位来使用，或者当页面没有滚动条时再用。
2.当使用一个独立的元素（或伪元素）来实现遮罩层时，这个遮罩层不仅可以从视觉上把用户的注意力引导到关键元素上，
还可以防止用户的鼠标与页面的其他部分发生交互，因为遮罩层会捕获所有指针事件。
box-shadow 并没有这种能力，因此它只能在视觉上起到引导注意力的作用，却无法阻止鼠标交互。
这一点是否可以接受，取决于你的具体需求。
