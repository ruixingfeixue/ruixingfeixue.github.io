css基础一

发表于 2017-05-12 | 分类于 css权威指南第三版

本书是css权威指南第三版,涵盖了CSS2和CSS2.1(截止2006年4月11日发布的工作草案)。

概念

1.替换元素
替换元素是指用来替换元素内容的部分并非由文档内容直接表示。
例如img元素，它是由文档本身之外的一个图像文件来替换。
<img src="a.png"这个标记片段不包含任何具体内容，只有一个元素名和属性。
只有指定了src，否则没有任何意义。还有input也是替换元素，其类型取决于type。
2.非替换元素
大部分的html和xhtml都是非替换元素。其内容在元素本身生成的框中显示。

属性

link标签：rel代表关系(relation).值为stylesheet。media属性，css2为这个属性定义了很多可取值。
all 用于表现所有媒体
aural 用于语音合成器、屏幕阅读器和文档的其它声音表现。
浏览器的支持性自己去查啦。毕竟用的也不多。
等等。。

选择器

1.分组选择器
h2,p{color:grey}这样就将h2和p分到一个组啦。h2和p颜色都是灰色啦。

2.多类选择器
.warning.urgent{color:grey} 这样就可以给元素增加样式class="warning urgent"。元素既紧急又有警告了。
.warning.help{color:red} 这个选择器只匹配class包含warning、help的元素。

3.属性选择器
p[class~="warning"]{font-weight:bold}; 这样就可以选择class属性中包含warning的元素。
[foo^="ch"] 选择foo属性值以”ch”开头的所有元素
[foo$="le"] 选择foo属性值以”le”结尾的所有元素
[foo*="l"] 选择foo属性值包含子串”l”的所有元素
[lang|='en'] 选择foo属性值等于en或者以en-开头的元素

4.伪类选择器
静态伪类：第一次显示之后不会再改变文档的样式，例如:link、:visited
动态伪类：例如:hover、:active、:focus
选择第一个子元素：:first-child来选择元素的第一个子元素

如代码，元素的第一个子元素包括第一个p，第一个em，第一个li。
p:first-child{ font-weight:bold;}
li:first-child{ text-transform:uppercase;}
第一个规则将作为某元素的第一个子元素的所有p元素设置为粗体。
第二个规则将作为某元素的第一个子元素额所有li元素变成大写。

5.伪元素选择器
:first-letter{color:red;} 用于设置一个块级元素首字母的样式。
:first-line{ color:red;} 用来影响元素中的第一个文本行。

注意点: :first-letter和:first-line只能运用于标记或者段落之类的块级元素，而不能运用于超链接等行内元素。

视口和屏幕尺寸

视口和屏幕尺寸不是同一视口是指浏览器窗口内的内容区域，不包含工具栏、标签栏等。也就是网页实际显示的区域。个概念。

屏幕尺寸指的是设备的物理显示区域。
需要注意的是有些浏览器调整工具显示的尺寸包含浏览器的其他元素，诸如地址栏、标签栏和搜索栏，而有些工具则不是这样。

媒体查询

<link rel="stylesheet" media="screen and (orientation: portrait)" href="portrait-screen.css" />
媒体查询表达式询问了媒体类型（你是一块显示屏吗？），然后询问了媒体特性（显示屏是纵向放置的吗？）。
任何纵向放置的显示屏设备都会加载 portrait-screen.css样式表，其他设备则会忽略该文件。
在媒体查询的开头追加not则会颠倒该查询的逻辑。

限制只有视口宽度大于800像素的显示屏设备才能加载文件。

有两点需要注意。
第一，媒体查询之间使用逗号分隔。
第二，你会注意到在projection之后，没有and，也没有任何特性/值的组合。
没有后续表达式，意味着只要是projection就满足条件。本例中，样式会应用于所有的投影仪。

在屏幕宽度小于等于400像素的设备上,h1元素的文字颜色就会变成绿色。
@import url("phone.css") screen and (max-width:360px);

CSS的@import指令在当前样式表中按条件引入其他样式表。
如上代码会给视口最大宽度为360像素的显示屏设备加载一个名为 phone.css的样式表。
切记，使用CSS的@import方式会增加HTTP请求(这会影响加载速度),所以请谨慎使用该方法。

穿衣搭配系列六-形态的动静

发表于 2017-04-27 | 分类于 穿衣搭配

之前我们学些了款式的量感，轮廓的直曲。再来复习下。
量感其实就是判断服饰的成熟度，服饰的直曲是女性化和中性化成都度量的指标，曲代表女性化程度很强，直代表偏中性化的服饰。
例如条纹格纹字母，抽象的图案。越接近我们自身的成熟度和女性化程度的衣服越适合自己。

今天我们来学习服饰形态的动静判断。其实就是视觉印象上的均衡与否，冲击力，表现性容。
形态的动静 也是从色彩、款式和面料三个纬度去判断

款式

动：时尚款，外形奇特不和常规，内部装饰多。与传统相对，有明显变化的款式混搭，多层次搭配，装饰明显，裁剪特殊。设计感强，不同款式间的搭配变化多。
静：基本款，外形常规，内部装饰图案少或者是无图案。传统款式，基本款式，简洁，均衡，变化少。装饰性弱。

分细节来说就是
外廓型：对服装结构的解构，异于常规的轮廓涉及（衬衣扣子设计是反的，跟常规不一样）
内部装饰：越抓住眼球越动（与流行装饰关系不大，例如铆钉、亮片）
图案：来源于图案本身、分散、细碎，不稳定，变化感强。而条纹格纹有稳定感。

说人话就是
服饰抓眼球、特别的程度。

色彩

纯度：也叫色彩的饱和度，是颜色的含色量。越鲜艳的颜色，纯度越高，色彩含量越高；
越柔和的颜色，纯度越低，色彩的含量月低。所有的无彩色都是纯度为0的颜色。

动：鲜艳，纯度高，对比强
静：纯度比较低，对比弱

Q：冬天着装偏深色，那在配饰或者打底上选择亮色或者浅色搭配，是不是就整体变为动了？
A：黑白配(极端颜色配比，一半一半的时候对比度最强),如果黑中一点点白色，那就不是动。量感直曲动静都是相互联系的

面料

动：反光，特殊肌理，高科技，面料之间有强烈的对比
静：亚光，常规，同一种面料缺少对比

注意：只要是色彩、面料、款式有一个方面是动的，那么这件衣服整体的形态就是动的。

直曲居中内装饰：小圆领
直曲居中图案：无图案
就算是直筒形的外廓，如果有荷叶边或者花朵藤蔓等女性感强的图案，也是偏曲。

3.色彩曲直和颜色的绝对冷暖的关系
绝对暖色:偏女性颜色 偏曲
绝对冷色：偏中性颜色 偏直

常用的hexo指令

发表于 2017-03-12 | 分类于 hexo

init

$ hexo init [folder]
新建一个网站。如果没有设置 folder ，Hexo 默认在目前的文件夹建立网站。

new

$ hexo new [layout] <title>
新建一篇文章。如果没有设置 layout 的话，默认使用 _config.yml 中的 default_layout 参数代替。如果标题包含空格的话，请使用引号括起来。

generate

$ hexo generate
生成静态文件。

选项 描述
-d, –deploy 文件生成后立即部署网站
-w, –watch 监视文件变动
该命令可以简写为

$ hexo g

publish

$ hexo publish [layout] <filename>
发表草稿。

server

$ hexo server
启动服务器。默认情况下，访问网址为： http://localhost:4000/。

选项 描述
-p, –port 重设端口
-s, –static 只使用静态文件
-l, –log 启动日记记录，使用覆盖记录格式

deploy

$ hexo deploy
部署网站。

参数 描述
-g, –generate 部署之前预先生成静态文件
该命令可以简写为：

$ hexo d

clean

$ hexo clean
清除缓存文件 (db.json) 和已生成的静态文件 (public)。
在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令。

version

$ hexo version
显示 Hexo 版本。

更多指令请参考 Hexo指令

自定义复选框

(Demo点这里)
替换元素： 据CSS2.1规范所述：“替换元素的特征在于，其内容超出了CSS格式化模型的范畴，比如图片、嵌入的文档或小应用程序等。”。
原则上我们无法为替换元素添加生成性内容，尽管某些浏览器可能会允许这样做。

由于label不是复选框那样的替换元素，我们可以为它添加生成性内容（伪元素），并基于复选框的状态来为其设置样式。
然后，就可以把真正的复选框隐藏起来（但不能把它从tab键切换焦点的队列中完全删除），再把生成性内容美化一番，用来顶替原来的复选框！

这个创意的名字叫做“复选框hack”

开关式按钮

(Demo点这里)
同样使用“复选框hack”的技巧
可以用来切换某个选项的开关状态：启用时，它是被按下的状态；停用时，它就是浮起的状态。
在使用开关式按钮时仍需慎重。在绝大多数场景下，开关式按钮对可用性有负面作用，
因为它们很容易与普通按钮混淆，让人误以为按下它会触发某个动作。

通过阴影来弱化背景

一、box-shadow方案
(Demo点这里)
使用大外延值实现 box-shadow: 0 0 0 999px rgba(0,0,0,.8);
这个初步的解决方案有一个明显的问题，就是它无法在较大的屏幕分辨率（>2000px）下正常工作。
视口单位: vmax。1vmax相当1vw和1vh两者中的较大值。
100vw等于整个视口的宽度，100vh就是视口的高度。因此，满足我们需求的最小值就是50vmax 。
由于投影是同时向四个方向扩展的，这个遮罩层的最终尺寸将是100vmax 加上元素本身的尺寸。
酱汁：box-shadow: 0 0 0 50vmax rgba(0,0,0,.8);

缺点：
1.由于遮罩层的尺寸是与视口相关，而不是与页面相关的，当我们滚动页面时，遮罩层的边缘就露出来了，
除非给它加上 position: fixed;这个样式，或者页面并没有长到需要滚动的程度。
此外，由于页面很可能真的很长，为了规避这个缺陷而扩大投影的扩张半径就不太明智了。
相反，我推荐有限度地应用这个技巧，比如配合固定定位来使用，或者当页面没有滚动条时再用。
2.当使用一个独立的元素（或伪元素）来实现遮罩层时，这个遮罩层不仅可以从视觉上把用户的注意力引导到关键元素上，
还可以防止用户的鼠标与页面的其他部分发生交互，因为遮罩层会捕获所有指针事件。
box-shadow 并没有这种能力，因此它只能在视觉上起到引导注意力的作用，却无法阻止鼠标交互。
这一点是否可以接受，取决于你的具体需求。

二、backdrop方案
(Demo点这里)
如果你想引导用户关注的元素就是一个模态的<dialog>元素
（<dialog>元素可以由它的 showModal()方法显示出来），
那么根据浏览器的默认样式，它会自带一个遮罩层。
借助 ::backdrop伪元素，这个原生的遮罩层也是可以设置样式的，比如可以把它变得更暗一些。

(Demo点这里)

每当弹出一个对话框，都需要给<main>元素增加一个类，以便对它应用模糊滤镜。

css字体排印

发表于 2016-12-30 | 分类于 css揭秘

字体排印

1.连字符断行
(Demo点这里)
text-align:justify;可以实现两端对齐。但是效果不佳。
hyphens: auto;可以解决长单词换行时的尴尬问题，使用-链接。
使用注意点：在 HTML 标签的lang属性中指定合适的语言，可以通过一些软连字符&shy;来辅助浏览器进行断词
但是目前没有任何浏览器支持。

2.插入换行
(Demo点这里)
有一个 Unicode 字符是专门代表换行符的： 0x000A。
在 CSS 中，这个字符可以写作 “\000A” ，或简化为 “\A” 。
我们可以用它来作为::after伪元素的内容，并将其添加到需要换行的元素尾部。

3.文本行的斑马条纹
(Demo点这里)
对背景设置统一的背景图像，再使用渐变生成背景图像。使用em来设定背景尺寸。这样可以适应font-size的变化。
注意点：background-size 需要设置为 line-height 的两倍，因为每个背景贴片需要覆盖两行代码。

4.调整tab的宽度
pre { tab-size: 2;} 可以调节缩进的宽度

5.连字
(Demo点这里)
font-variant 被升级成了一个简写属性，由很多新的展开式属性组合而成。
其中font-variant-ligatures专门用来控制连字效果的开启和关闭。
如果要启用所有可能的连字，需要同时指定这三个标识符：
font-variant-ligatures: common-ligatures discretionary-ligatures historical-ligatures;

6.华丽的&符号
通常会在 font-family 声明中同时指定多个字体（即字体队列）。
这样，即使我们指定的最优先字体不可用，浏览器还可以回退到其他符合整体设计风格的字体。
但是，很多开发者都忽略了一点：这个机制对单个字符来说也是有效的。
如果某款字体可用，但仅包括某几个字符，那它就只会用来显示这几个字符；
而在显示其他字符时，浏览器就会回退到其他字体。这个规则对本地字体和通过@font-face规则引入的嵌入字体都是有效的。

css视觉效果

发表于 2016-12-14 | 分类于 css揭秘

单侧投影

(Demo点这里)
1.单边投影
对元素使用4px的模糊半径意味着投影的尺寸会比元素本身的尺寸大约8px。这个如何解决呢?
最终的解决方案来自 box-shadow 鲜为人知的第四个长度参数。它排在模糊半径参数之后，称作扩张半径。
这个参数会根据你指定的值去扩大或(当指定负值时)缩小投影的尺寸。
举例来说，一个-5px的扩张半径会把投影的宽度和高度各减少10px(即每边各5px)。
辣么把扩张半径设为负的模糊值不就看不见投影啦。再用偏移量来移动，不就形成单边投影啦。

2.邻边投影
技巧1:我们不应该把投影缩得太小，而是只需把阴影藏进一侧，另一侧自然露出就好。因此，扩张半径不应设为模糊半径的相反值，而应该是这个相反值的一半。
技巧2: 需要指定两个偏移量，因为我们希望投影在水平和垂直方向上同时移动。它们的值需要大于或等于模糊半径的一半，因为我们希望把投影藏进另外两条边之内。

3.双侧投影
因为扩张半径在四个方向上的作用是均等的（也就是说，我们无法指定投影在水平方向上放大，而在垂直方向上缩小）
唯一的办法是用两块投影（每边各一块）来达到目的。然后基本上就是把“单侧投影”中的技巧运用两次。

不规则投影

(Demo点这里)
1.box-shadow不支持不规则投影。（元素添加了一些伪元素或半透明的装饰之后）具体情形如下：
半透明图像、背景图像、或者 border-image （比如老式的金质像框）;
元素设置了点状、虚线或半透明的边框，但没有背景（或者当background-clip 不是 border-box 时）;
对话气泡，它的小尾巴通常是用伪元素生成的;
我们在“切角效果”一节中见过的切角形状;
几乎所有的折角效果，包括“折角效果”一节将提到的例子;
通过 clip-path 生成的形状，比如“菱形图片”一节中提到的菱形图像。

2、解决方案 （使用CSS滤镜）
例如drop-shadow() 滤镜可接受的参数基本上跟 box-shadow 属性是一样的，但不包括扩张半径，不包括 inset 关键字，也不支持逗号分割的多层投
影语法。
举个例子，box-shadow: 2px 2px 10px rgba(0,0,0,.5); 可以这样写 filter: drop-shadow(2px 2px 10px rgba(0,0,0,.5));
CSS 滤镜最大的好处：它们可以平稳退化：当浏览器不支持时，不会出现问题，只不过没有任何效果而已。
如果你确实需要这个效果在尽可能多的浏览器中显示出来，可以同时附上一个 SVG 滤镜，这样可以得到稍微好一些的浏览器支持度。

染色效果

(Demo点这里)
1.使用滤镜
sepia()会给图片增加一种降饱和度的橙黄色染色效果，几乎所有像素的色相值会被收敛到 35~40。
如果我们想要的主色调的饱和度比这更高，可以用saturate()滤镜来给每个像素提升饱和度。
hue-roatate()滤镜，把每个像素的色相以指定的度数进行偏移。为了把原有的色相值40改变为335，需要增加大约295度(335-40)。
2.基于混合模式的方案
“混合模式”控制了上层元素的颜色与下层颜色进行混合的方式。
用它来实现染色效果时，需要用到的混合模式是luminosity。
这种luminosity混合模式会保留上层元素的HSL亮度信息，并从它的下层吸取色相和饱和度信息。
要对一个元素设置混合模式，有两个属性可以派上用场：
mix-blend-mode可以为整个元素设置混合模式，background-blend-mode可以为每层背景单独指定混合模式。
使用这种策略有两种选择模式：
第一种选择：需要把图片包裹在一个容器中，并把容器的背景色设置为我们想要的主色调。
第二种选择：不用图片元素，而是用div元素——把这个元素的第一层背景设置为要染色的图片，并把第二层的背景设置为我们想要的主色调。
